Importando um projeto para o ambiente

Para importar um projeto é necessário compactar, em um arquivo .zip, o diretório raiz com todos os arquivos do seu projeto. Por exemplo, se o diretório raiz do seu projeto for /home/fulano/projetos/meuMaisNovoProjeto/, então o diretório meuMaisNovoProjeto precisará ser interiramente compactado em um arquivo zip (o diretório meuMaisNovoProjeto deverá estar na raiz do arquivo .zip). A organização recomendada para a estrutura do seu projeto é mostrada na imagem abaixo. Essa organização deve ser seguida para que seja possível executar o projeto através do navegador. O tamanho máximo permitido para o arquivo .zip é de 50Mb.

Perceba que no diretório src apenas serão aceitos arquivos .java (arquivos diferente disso serão ignorados). De forma análoga, no diretório libs apenas serão aceitos arquivos .jar. Já no caso do diretório resources é possível enviar qualquer tipo de arquivo. Além dos arquivos mostrados na imagem acima, também serão considerados arquivos de configuração de algumas IDEs. As IDEs suportadas são Eclipse e NetBeans, mas caso você utilize alguma outra IDE, basta entrar em contato para que ela também seja suportada por este ambiente.

Exportanto um projeto do Eclipse para um arquivo Zip

  1. Vá no menu File > Export...
  2. Selecione a opção General > Archive File.
  3. Selecione o projeto que deseja exportar.
  4. Marque a caixa de seleção "Save in zip format".
  5. Escolha um local onde o arquivo .zip será salvo.
  6. Clique em Finish.

Utilizando imagens ao executar pelo Applet

Siga os seguintes passos para que possa utilizar imagens em seu programa de modo que essas imagens sejam exibidas através dos applets:

  1. Crie um .jar contendo as imagens que você deseja utilizar (as imagens deverão estar dentro de um pacote qualquer).
  2. Inclua o .jar na pasta libs do seu projeto.
  3. Para obter o endereço da imagem e poder usá-la, faça: getClass().getResource("/pacoteOndeEstaAImagem/imagem.jpg");.
  4. Projeto com exemplo de como fazer.

Codificação UTF-8 problema de compilação

Se você utiliza Windows, provavelmente seus arquivos Java estarão compactados com uma codificação diferente de UTF-8. Assim, ao tentar compilar seu programa, não será possível, pois o servidor espera que seus arquivos estejam codificados com UTF-8.

Para resolver esse problema, você precisa configurar o seu editor para que os arquivos Java sejam salvos como UTF-8. Perceba que ao alterar a configuração do seu editor, é possível que alguns caracteres não tenham sidos convertidos corretamente. Nesses casos, você ainda continuará impossibilitado de compilar o projeto e deverá alterar os caracteres inválidos manualmente. Para evitar esse problema, sempre que iniciar um projeto, configure o seu editor para utilizar UTF-8.

Para configurar o Eclipse para utilizar UTF-8 no seu projeto, siga os seguintes passos:

  1. Clique com o botão direito no seu projeto.
  2. Selecione a opção Properties.
  3. Selecione Resource.
  4. Habilite a caixa de seleção Other.
  5. Selecione a opção UTF-8.
  6. Clique em Apply e depois em OK.

Bibliotecas faltando problema de compilação

Para que seu projeto possa ser compilado é necessário incluir na pasta libs, na raiz do seu projeto, todas as bibliotecas do qual o seu projeto depende. Isso deverá ser feito também para as bibliotecas JUnit e Hamcrest. Perceba que algumas IDEs gerenciam essas bibliotecas automaticamente para você. Por isso, se você não incluir essas bibliotecas na pasta libs, o seu projeto não compilará através do ambiente (mesmo que esteja compilando localmente em seu computador). Você pode baixar as bibliotecas JUnit e Hamcrest nesse projeto.

Não esqueça que os arquivos .jar com suas bibliotecas devem estar compilados e devem conter os arquivos .class. Caso você envie um arquivo .jar com arquivos .java, então a compilação no servidor não irá funcionar.

Fechar janelas do Swing problema de execução

Para que seu projeto possa ser executado através de um applet é necessário que ele respeite as restrições impostas para applets. Programas rodando em applets não possuem permissão para encerrar a máquina virtual. A opção JFrame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE), faz justamente isso, encerra a máquina virtual. Para resolver esse problema e conseguir executar o seu projeto através do ambiente Web, utilize a opção JFrame.setDefaultCloseOperation(JFrame.DISPOSE_ON_CLOSE). Através desse método a janela de sua interface gráfica será eliminada e o fluxo de execução seguirá normalmente até que o seu programa seja encerrado naturalmente (o que deve acontecer logo após a sua janela ser fechada).

Classe sem o método main problema de execução

Seu projeto somente será executado corretamente através do applet se a classe que você definiu como principal no momento da compilação através do ambiente Web tiver um método estático chamado main. A assinatura do seu método main deve ser a seguinte public static void main(String[] argumentos).

Versão do Java problema de compilação e execução

Ao tentar compilar um projeto através deste ambiente Web você deverá se certificar que seu projeto compila com a versão 8 do Java. Além disso, para executar o seu projeto através do navegador é necessário que você tenha instalado o Java 8 ou superior. Outro fator importante é que navegadores como o Google Chrome abandonaram o suporte aos applets. Dessa forma, dependendo da sua versão do navegador, os applets poderão não funcionar. Até o momento, o navegador com melhor suporte aos applets é o Firefox.

Se você usa Ubuntu e o Java 8 não estiver disponível nos repositórios oficiais, então poderá instalar o Java 8 através desse PPA.
